# Book 01-06 - Remote Work Ecosystem

**Version:** v1 draft
**Status:** Draft
**Document ID:** MAL-V01-B006


## Purpose
Define how Mariam supports distributed teams and remote execution.

## Scope
Covers async coordination, handoffs, accountability, knowledge continuity, and operating rhythm.

## Principles
- Remote work requires explicit context.
- Async execution depends on durable decisions and clear ownership.
- Coordination tools should reduce status noise.

## Operating Rules
- Record decisions, blockers, handoffs, and next actions in structured forms.
- Design workflows for time-zone tolerant collaboration.
- Expose accountability without creating surveillance behavior.

## Acceptance Criteria
- Remote strategy traces to governance and quality principles.
- Future workflows can support async review and handoff.
- The system distinguishes accountability from micromanagement.
