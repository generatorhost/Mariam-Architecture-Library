# Volume 00 Acceptance

**Version:** v1 draft
**Status:** Draft
**Document ID:** MAL-V00-ACCEPTANCE


## Purpose
This document defines the acceptance rules for treating Volume 00 as a usable constitutional baseline.

## Scope
It covers document completeness, review readiness, release packaging, and readiness for downstream volumes.

## Principles
- A constitutional release must be coherent before it is comprehensive.
- Acceptance is based on usable governance value, not page count.
- Draft status still requires internal consistency.

## Operating Rules
- Each book must include purpose, scope, principles, operating rules, acceptance criteria, and version history.
- Release artifacts must include Markdown, PDF, ZIP, manifest, and release notes.
- Missing downstream volumes are acceptable only when clearly reserved.

## Acceptance Criteria
- Volume 00 contains README, INDEX, CHANGELOG, TRACEABILITY, ACCEPTANCE, and 15 books.
- Generated PDF includes every Volume 00 Markdown file in deterministic order.
- Manifest records all generated release artifacts with size and checksum.
