# Book 00-12 - Engineering Principles

**Version:** v1 draft
**Status:** Draft
**Document ID:** MAL-V00-B012


## Purpose
Define engineering behavior expected when Mariam implementation begins.

## Scope
Covers architecture discipline, code quality, testing, review, automation, and documentation alignment.

## Principles
- Engineering follows documented architecture.
- Small, reversible changes are preferred.
- Automated verification supports human review.

## Operating Rules
- Do not implement features without a traceable specification or approved issue.
- Keep module boundaries aligned with blueprint ownership.
- Update documentation when implementation reveals a valid architectural change.

## Acceptance Criteria
- Engineering standards can expand from this baseline.
- Pull requests cite relevant documents.
- Implementation work does not outrun approved architecture.
