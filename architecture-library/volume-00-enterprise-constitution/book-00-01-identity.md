# Book 00-01 - Identity

**Version:** v1 draft
**Status:** Draft
**Document ID:** MAL-V00-B001


## Purpose
Define Mariam AI Enterprise OS as a governed enterprise operating system for AI-enabled work, not a loose collection of tools.

## Scope
Covers naming, identity boundaries, product posture, and the relationship between documentation and implementation.

## Principles
- Mariam is an enterprise OS for orchestrating AI, human work, knowledge, governance, and execution.
- The system identity is durable even when modules, vendors, or implementation technologies change.
- Documentation is part of the product, not an accessory.

## Operating Rules
- Use the full name in constitutional and release contexts.
- Do not redefine Mariam as a chatbot, single app, marketplace only, or automation script.
- Reject features that weaken the enterprise operating-system identity.

## Acceptance Criteria
- A contributor can state what Mariam is in one paragraph.
- The repository README identifies this library as the Source of Truth.
- Future volumes can reference identity without ambiguity.
