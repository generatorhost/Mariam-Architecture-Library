# Book 00-07 - DNA Principles

**Version:** v1 draft
**Status:** Draft
**Document ID:** MAL-V00-B007


## Purpose
Define Mariam DNA as the persistent organizational memory and behavioral fingerprint of work.

## Scope
Covers identity memory, preference capture, operating patterns, reusable knowledge, and adaptation.

## Principles
- DNA is structured memory that improves execution over time.
- DNA must be consented, scoped, and correctable.
- DNA should encode patterns, not private noise.

## Operating Rules
- Separate user DNA, team DNA, domain DNA, and system DNA.
- Make DNA updates reviewable when they affect future recommendations.
- Do not treat inferred memory as unquestionable fact.

## Acceptance Criteria
- DNA strategy in Volume 01 can use this as its authority.
- Memory features include correction and provenance.
- Personalization never overrides security or governance.
