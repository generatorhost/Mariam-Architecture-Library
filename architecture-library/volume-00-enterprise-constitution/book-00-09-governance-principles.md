# Book 00-09 - Governance Principles

**Version:** v1 draft
**Status:** Draft
**Document ID:** MAL-V00-B009


## Purpose
Define decision authority, review expectations, and change control for the enterprise OS.

## Scope
Covers policy, approval, ownership, exceptions, and documentation authority.

## Principles
- Governance should enable confident speed.
- Authority must be explicit.
- Exceptions expire unless renewed.

## Operating Rules
- Assign owners to constitutional, roadmap, blueprint, and specification decisions.
- Document policy exceptions with rationale and review date.
- Treat undocumented decisions as provisional.

## Acceptance Criteria
- Release notes identify governance impact.
- Roadmap changes can cite approval authority.
- Contributors know where decisions belong.
