# Book 00-14 - Development Lifecycle

**Version:** v1 draft
**Status:** Draft
**Document ID:** MAL-V00-B014


## Purpose
Define the lifecycle from idea to release for Mariam work.

## Scope
Covers discovery, documentation, design, implementation, verification, release, operation, and feedback.

## Principles
- Documentation leads implementation.
- Acceptance criteria are written before delivery claims.
- Feedback becomes structured improvement, not scattered memory.

## Operating Rules
- Start significant work with a document, issue, or specification reference.
- Move from draft to approved status through review.
- Do not release without manifest, notes, and verification evidence.

## Acceptance Criteria
- Roadmap items can be staged through lifecycle states.
- Release artifacts demonstrate completed lifecycle steps.
- Contributors understand when work is ready to build.
