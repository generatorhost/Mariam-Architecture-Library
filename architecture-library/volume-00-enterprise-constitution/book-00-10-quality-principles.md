# Book 00-10 - Quality Principles

**Version:** v1 draft
**Status:** Draft
**Document ID:** MAL-V00-B010


## Purpose
Define what quality means for Mariam before implementation details exist.

## Scope
Covers correctness, usability, resilience, observability, accessibility, and maintainability.

## Principles
- Quality is measured across product behavior, documentation, operations, and user trust.
- A workflow is not complete until failure behavior is designed.
- Usability and reliability are architectural concerns.

## Operating Rules
- Each specification must define success and failure behavior.
- Testing must cover permissions, state transitions, and recovery paths.
- Documentation must be clear enough to guide implementation.

## Acceptance Criteria
- Testing guide can derive checks from this book.
- Acceptance criteria are present for release artifacts.
- Quality risks are recorded before implementation.
