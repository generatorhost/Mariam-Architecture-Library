# Book 00-04 - Core Principles

**Version:** v1 draft
**Status:** Draft
**Document ID:** MAL-V00-B004


## Purpose
Declare the cross-cutting principles that govern every Mariam decision.

## Scope
Covers simplicity, explicitness, traceability, modularity, accountability, and user trust.

## Principles
- Make important state visible.
- Design modules with clear boundaries.
- Prefer boring reliability over impressive fragility.
- Treat governance as a user experience.

## Operating Rules
- No critical workflow may depend on hidden assumptions.
- Every module must have a defined responsibility.
- Complexity must earn its place through measurable value.

## Acceptance Criteria
- Architecture reviews can cite core principles.
- Unclear ownership is flagged before implementation.
- Product behavior remains understandable to operators.
