# Book 02-05 - Phase 03 AI Society

**Version:** v2 draft
**Status:** Draft
**Document ID:** MAL-V02-B005


## Purpose
Define the first governed society of AI agents and assistants within Mariam.

## Scope
Covers agent roles, boundaries, collaboration contracts, supervision, evaluation, and escalation.

## Strategic Objective
Create a controlled multi-agent layer that supports enterprise work without hidden autonomous authority.

## Systems Included
- Agent registry
- Agent role definitions
- Prompt and context governance
- Evaluation harness
- Escalation workflows

## Main Deliverables
- AI society architecture
- Agent contract template
- Prompt governance rules
- Evaluation scenarios
- Human review checkpoints

## Dependencies
- Enterprise core permissions and audit events.
- AI principles and swarm principles.
- Knowledge strategy for context provenance.

## Risks
- Agents may appear more authoritative than their actual evaluation supports.
- Prompt drift may weaken repeatability.
- Escalation paths may be skipped if UX is too slow.

## Acceptance Criteria
- Every agent has a bounded role and output contract.
- High-impact outputs are reviewable.
- Agent actions are traceable to context and user intent.

## Implementation Notes
- Start with few high-value agents before adding broad agent catalogs.
- Separate drafting agents from execution agents.
- Keep model provider details outside the agent identity.
