# Book 02-15 - Phase 13 Remote Work Automation

**Version:** v2 draft
**Status:** Draft
**Document ID:** MAL-V02-B015


## Purpose
Define automation for distributed teams, async coordination, handoffs, updates, and accountability.

## Scope
Covers async summaries, decision records, blockers, handoffs, meeting-to-workflow conversion, and operating rhythm support.

## Strategic Objective
Make remote work clearer and less dependent on meetings or scattered chat history.

## Systems Included
- Async update engine
- Decision record automation
- Blocker tracking
- Handoff workflows
- Remote operating dashboards

## Main Deliverables
- Remote work automation specification
- Async summary templates
- Decision record schema
- Blocker lifecycle
- Team rhythm playbooks

## Dependencies
- Enterprise organization model.
- Knowledge system.
- Remote work strategy.

## Risks
- Automation can become surveillance if framed poorly.
- Summaries may omit important nuance.
- Async workflows may clash with existing team habits.

## Acceptance Criteria
- Remote workflows preserve decisions, blockers, and owners.
- Users can review generated summaries.
- Operating dashboards focus on work state, not personal monitoring.

## Implementation Notes
- Design for trust and clarity.
- Let teams tune cadence and notification intensity.
- Use knowledge provenance for summaries.
