# Book 03.02.11 - Cross Department Coordination

**Version:** v4 draft
**Status:** Draft
**Document ID:** MAL-V03-B03-02-011

## Purpose
Define coordination patterns between departments for shared missions, handoffs, dependencies, and joint reviews.

## Scope
Covers coordination groups, cross-functional missions, handoff protocols, dependency tracking, and shared accountability.

## Responsibilities
- Create coordination structures.
- Track cross-department dependencies.
- Define handoff responsibilities.
- Support joint operating reviews.

## Inputs
- Department missions.
- Dependency request.
- Handoff context.
- Shared KPI target.
- Coordination charter.

## Outputs
- Coordination record.
- Dependency map.
- Handoff record.
- Joint review outcome.

## Interfaces
- Department missions.
- Operating rhythm.
- Workflow system.
- Knowledge system.

## Events
- CoordinationGroupCreated
- DependencyLogged
- HandoffAccepted
- JointReviewCompleted

## Storage
- Coordination group store.
- Dependency ledger.
- Handoff history.

## Security
- Shared records respect participating department permissions.
- Sensitive dependencies are classified.
- Handoff acceptance is auditable.

## Metrics
- Dependency resolution time.
- Handoff acceptance rate.
- Joint review completion.
- Cross-department blocker count.

## Tests
- Coordination group tests.
- Dependency lifecycle tests.
- Handoff authorization tests.
- Joint review tests.

## Acceptance Criteria
- Cross-department work has owners and handoffs.
- Dependencies are visible.
- Joint reviews create actionable outcomes.
