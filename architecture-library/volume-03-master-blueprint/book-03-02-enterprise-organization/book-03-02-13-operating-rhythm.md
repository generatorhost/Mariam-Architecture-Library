# Book 03.02.13 - Operating Rhythm

**Version:** v4 draft
**Status:** Draft
**Document ID:** MAL-V03-B03-02-013

## Purpose
Define the cadence of enterprise operations: reviews, planning cycles, retrospectives, board sessions, and execution checkpoints.

## Scope
Covers recurring meetings, async updates, review packs, decision windows, and rhythm-driven accountability.

## Responsibilities
- Define operating cadences.
- Schedule reviews and checkpoints.
- Collect review inputs.
- Convert review outcomes into actions.

## Inputs
- KPI status.
- Department reports.
- Escalations.
- Decision backlog.
- Board and chief office calendars.

## Outputs
- Operating calendar.
- Review packet.
- Action register.
- Rhythm completion events.

## Interfaces
- Enterprise KPIs.
- Chief office.
- Board governance.
- Remote work automation.

## Events
- OperatingReviewScheduled
- ReviewPacketSubmitted
- OperatingReviewCompleted
- ActionRegistered

## Storage
- Operating calendar store.
- Review packet archive.
- Action register.

## Security
- Review packets may contain sensitive enterprise data.
- Access follows review membership and role.
- Action records retain accountability.

## Metrics
- Review completion rate.
- Packet freshness.
- Action closure rate.
- Missed cadence count.

## Tests
- Cadence scheduling tests.
- Review packet access tests.
- Action lifecycle tests.

## Acceptance Criteria
- Operating rhythm is explicit and repeatable.
- Reviews produce owned actions.
- Cadence records can be audited.
